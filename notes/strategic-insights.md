# Workflows

> Kevin Indig–derived workflows for AI-powered SEO content production.

## 1. Research-Led Content Strategy

```
Research question → Partner for data (Clickstream / Omnia / Surfer)
    → Analyze (UX study / clickstream / citation audit)
    → Publish findings (Growth Memo model)
    → Extract operator guidance
    → Build micro-tools for premium/internal use
```

**When to use:** Establishing thought leadership or validating a new SEO thesis before scaling production.

---

## 2. Sales-Call → Content Pipeline (Ramp Model)

```
Filter won sales conversations
    → Transcribe (Gong / manual)
    → AI extract via Humata or similar (pain, objections, alternatives, success criteria)
    → Identify topics (ignore search volume at first)
    → Create content addressing extracted questions
    → Measure conversion lift (not organic traffic)
```

**When to use:** B2B SaaS with recorded sales calls and long sales cycles.

---

## 3. Partial-Automation Content Production

```
Codify workflow:
  ideation → brief → draft → edit → schema markup → publish → distribute → refresh

Identify bottleneck step (usually draft or SERP metadata)
    → Automate bottleneck with AirOps / Claude
    → Human review: facts, brand voice, YMYL compliance
    → Quarterly refresh for citation freshness
```

**When to use:** Teams scaling content output without adding headcount.

---

## 4. Off-Site Citation Building

```
Map commercial prompts in your category
    → Audit which third-party sources LLMs cite (Reddit, G2, publishers)
    → Execute: digital PR, community threads, YouTube walkthroughs, review campaigns
    → Track mentions and citations per engine separately
    → Iterate based on presence/portability/concentration metrics
```

**When to use:** On-page optimization plateau; competitors cited despite weaker site content.

---

## 5. Interactive Tool Production (Vibe Coding)

```
Identify high-intent page cluster
    → Write PRD with AI (problem, inputs, outputs, embed plan)
    → Vibe-code MVP (Cursor / v0 / Lovable)
    → Validate engagement on staging
    → Hand to dev for production hardening
    → Embed across content hub pages
```

**When to use:** Zero-click SERPs where text answers are synthesized without site visits.

---

## 6. SERP Second-Impression Optimization

```
Identify pages with AIO presence but low CTR
    → Audit competitor SERP snippets sharing the AIO
    → Apply page-type template:
        PDP: Product schema (aggregateRating, review, offers, availability)
        CDP: ItemList schema, facet sitelinks, depth signals
        Blog: dateModified, Article schema with author sameAs entity
    → Rewrite title/meta for re-read comparison (Claude skill or manual)
    → Test and iterate
```

**When to use:** High impressions, collapsing clicks, AIO on SERP.

---

## 7. AI Visibility Measurement Loop

```
Define prompt set (commercial + informational, per market)
    → Query each engine 5–10× per prompt (probabilistic)
    → Calculate: presence, portability, concentration
    → Identify engine-specific gaps
    → Assign production/distribution fixes per lane
    → Re-measure monthly
```

**When to use:** Replacing rank-tracker mindset for AI surfaces.
